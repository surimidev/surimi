      // SPA transitions need a JS trigger. Surimi only writes the CSS above.
      const card = document.querySelector('.card');
      let alt = false;
      function swap() {
        alt = !alt;
        const apply = () => {
          card.textContent = alt ? 'New state!' : 'I morph between states.';
          card.style.background = alt ? '#3498db' : '#eee';
          card.style.color = alt ? 'white' : 'black';

        if (document.startViewTransition) {
          document.startViewTransition(apply);

          apply();
        }
      }

## Name once, style the snapshots

`viewTransition('card')` returns a token that is the view transition name. Assign it to an element with `use()` (or `style({ viewTransitionName: card })`), then style the four snapshot pseudo-elements the browser creates:

```ts
const card = viewTransition('card');

select('.card').use(card); // .card { view-transition-name: card }

card
  .old({ animation: `${fadeOut} 0.3s` })   // ::view-transition-old(card)
  .new({ animation: `${fadeIn} 0.3s` })    // ::view-transition-new(card)
  .group({ animationDuration: '0.4s' });   // ::view-transition-group(card)
```

The big gotcha this hides: those `::view-transition-*` rules live on the document `:root` overlay, **not** on `.card`. The builder always emits them at the top level and keeps the `(card)` name in sync for you.

## Old vs new

The default browser animation is a cross-fade. To make enter and exit differ, put the exit on `old()` and the entrance on `new()` — they map to the outgoing and incoming snapshots. Animate `group()` for the size/position morph between the two.

## Targeting many at once

Share a `view-transition-class` across elements and style the whole bucket with one rule:

```ts
select('.a', '.b').style({ viewTransitionClass: 'card' });

viewTransitionClass('card').group({ animationDuration: '0.4s' }); // ::view-transition-group(*.card)
viewTransition.all().old({ animation: 'none' });                  // ::view-transition-group(*)
```

## CSS only

Surimi emits CSS, never JavaScript. Cross-document (MPA) transitions work from CSS alone via `viewTransition.navigation('auto')`. Same-document (SPA) transitions still need a `document.startViewTransition(() => { ... })` call in your own code to trigger the snapshot — the CSS here just defines how it animates.
