## Style B when A happens

CSS can't normally say "style x when y is hovered" across the tree. Surimi's `when()` function can, using `:has()` under the hood.

```ts
import { when } from 'surimi/conditional';

when('.trigger').hovered().select('.target').style({
  background: '#3498db',
});
// -> :where(html):has(.trigger:hover) .target { background: #3498db; }
```

The condition methods read like English: `hovered()`, `focused()`, `checked()`, `disabled()`, and so on. Chain them:

```ts
when('.input').focused().and.not.disabled().select('.hint').style({ opacity: 1 });
when('.a').hovered().or.focused().select('.b').style({ /* ... */ });
```

> [!NOTE]
> `when()` lives in `surimi/conditional`, a separate import from the main package.
