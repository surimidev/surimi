## Real, typed variables

`property()` registers a CSS [`@property`](https://developer.mozilla.org/en-US/docs/Web/CSS/@property) and gives you back a token you can drop in as a value. It compiles to `var(--name)`.

```ts
const primary = property('primary', '#3498db', '<color>');

select('.btn').style({ background: primary });
// -> .btn { background: var(--primary); }
```

The third argument is the [syntax](https://developer.mozilla.org/en-US/docs/Web/CSS/@property/syntax), so the browser knows the type (and can even animate it). Change `#3498db` above and the button follows.

## Theming with modes

For light/dark (or any named modes), use `createTheme` from `surimi/theme`. It registers tokens once, assigns values per mode, and returns typed refs:

```ts
import { createTheme } from 'surimi/theme';

const theme = createTheme({
  modes: {
    light: ':root',
    dark: '[data-theme="dark"]',
  },
  tokens: {
    bg: { app: { light: '#fff', dark: '#111', syntax: '<color>' } },
    text: { default: { light: '#111', dark: '#eee' } },
  },
});

select('body').style({ background: theme.bg.app, color: theme.text.default });
```

Targets can be selectors, attribute toggles, or media queries — fully configurable per mode.

In the preview iframe, flip modes by setting `data-theme` on the document root:

```html
<script>
  document.documentElement.setAttribute('data-theme', 'dark');
</script>
```

The example above wires that to a **Toggle dark mode** button so you can see light/dark tokens swap live.

## The superpower: it's just TypeScript

Group tokens in an object, export them from their own file, and import them anywhere — fully typed:

```ts
// theme.css.ts
export const theme = createTheme({ modes: { light: ':root' }, tokens: { ... } });
```

(In this single-file playground we keep them inline, but in a real project each `.css.ts` emits its own CSS and shares the tokens through normal imports.)
